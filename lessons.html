01. Core Electronics

Beginner

Ohm's Law Masterclass

The universal circuit equation V = I × R governs ALL electrical behavior. Every LED resistor, power supply, and sensor circuit depends on this relationship.

Core Skills You'll Master:

  • Calculate exact resistor values: R = (Vsupply - Vled) / Iled
  • Series: Rtotal = R1 + R2 + R3, Current constant
  • Parallel: 1/Rtotal = 1/R1 + 1/R2, Voltage constant
  • Power: P = V × I prevents resistor burnout
Pro Insight:

5V Arduino pin + 20mA LED (2V drop) = 150Ω resistor. Use 180Ω (nearest E24 value) for safety. Never guess—always calculate!

Beginner

Component Anatomy

90% of debugging fails due to misidentification. Master visual recognition and markings to instantly know component values and polarity.

Identification Mastery:

  • Resistors: Brown-Black-Red-Gold = 1kΩ ±5%
  • Capacitors: 104 = 100nF, 476 = 47µF
  • Diodes: Band = Cathode, LED longer leg = Anode
  • Transistors: C-B-E pinout, not all packages same
Debug Trick:

Multimeter diode mode beeps for forward bias only. Test ALL pins systematically. No beep = dead component or wrong polarity.

Beginner

Power Supply Design

Arduino USB power drops under load. Build reliable 5V/3.3V rails that won't crash your project mid-operation.

Bulletproof Power:

  • 7805 needs 7V-33V in → 5V out, 1A max
  • 100nF + 10µF caps = ripple-free output
  • Heat calculation: Pd = (Vin - 5V) × Iload
  • Reverse polarity diode prevents magic smoke
Thermal Reality:

12V→5V at 500mA = 3.5W heat! Without heatsink, 7805 hits 150°C in 30 seconds. Always calculate dissipation first.

Intermediate

Kirchhoff's Circuit Laws

Solve ANY circuit without simulation. KVL + KCL = complete current/voltage analysis toolkit.

Mathematical Precision:

  • KVL: Sum of voltages around loop = 0V
  • KCL: Sum of currents into node = 0A
  • Voltage divider: Vout = Vin × (R2/(R1+R2))
  • Current divider: I1 = Iin × (R2/(R1+R2))
Real Circuit Hack:

ADC reference needs 1.1V? Use 10k+10k divider from 5V = 2.5V, then another divider for precision. Measure, don't assume!

02. Arduino Programming

Intermediate

Setup() & Loop() Architecture

95% of Arduino bugs come from poor structure. Master the ONE correct way to organize microcontroller code.

Production Code Structure:

  • setup(): One-time pin config, serial init
  • loop(): Non-blocking main program
  • Global variables for shared state only
  • Functions for reusable code blocks
Common Trap:

delay(1000) in loop() = unresponsive robot. Use millis() timing instead. Non-blocking = professional.

Intermediate

Digital vs Analog Signals

Arduino speaks binary (digital) but hears analog world. Bridge the gap with proper signal conversion.

Pin Functions Decoded:

  • digitalRead(pin) → HIGH/LOW (5V/0V)
  • analogRead(A0) → 0-1023 (10-bit ADC)
  • analogWrite(pin, 128) → PWM 50% duty (pins 3,5,6,9,10,11)
  • map(value, 0,1023, 0,255) scales ranges
Sensor Reality:

Potentiometer 0-5V → analogRead() → 0-1023 → map() → 0-100%. Photoresistor needs 10k pull-down for full range.

Advanced

I²C Multi-Device Protocol

Connect 127 sensors to 2 Arduino pins. I²C turns single-board into multi-device powerhouse.

I²C Implementation:

  • #include <Wire.h>; Wire.begin();
  • MPU6050 = 0x68, BMP280 = 0x76 addresses
  • Wire.write(reg); Wire.requestFrom(addr, bytes);
  • I²C scanner finds all devices automatically
Pro Tip:

4.7kΩ pullups mandatory! No pullups = no communication. 400kHz speed max for 100kHz reliability with long wires.

Advanced

SPI High-Speed Protocol

10x faster than I²C. Perfect for displays, SD cards, fast sensors requiring real-time data.

SPI Hardware Pins:

  • Uno: 11(MOSI), 12(MISO), 13(SCK)
  • Each device needs unique CS pin
  • 4 clock modes (CPOL/CPHA) match device specs
  • SPI.beginTransaction(SPISettings(1e6, MSBFIRST, SPI_MODE0))
Speed Reality:

OLED display updates @ 8MHz = buttery smooth. SD card logging hits 20MB/s bursts. Wrong clock mode = garbage data.

Advanced

Interrupts & Hardware Timers

Replace wasteful polling with instant event response. 1000x more efficient real-time code.

Event-Driven Code:

  • attachInterrupt(digitalPinToInterrupt(2), ISR, RISING)
  • Timer1: 16-bit, up to 62.5ns resolution
  • ISR must execute <10µs, no delay()/Serial
  • Flag variables communicate ISR → loop()
Performance Leap:

Encoder reads: polling=50% CPU, interrupt=0.1% CPU. 50kHz PWM generation without blocking main loop.

03. Sensors & Actuators

Intermediate

Temperature Sensing Systems

Three sensor types, one interface standard. Reliable temperature measurement for any environment.

Sensor Arsenal:

  • DHT22: ±0.5°C, humidity bonus
  • DS18B20: 1-Wire, waterproof probes
  • NTC thermistor: Cheap, needs linearization
  • Multiple sensors on single pin (parasitic power)
Data Quality:

Raw NTC readings nonlinear. Use Steinhart-Hart equation or lookup table. 10-sample average kills noise.

Intermediate

Servo & Stepper Precision

Convert Arduino pulses to mechanical motion. Foundation of robotics and automation.

Motion Control:

  • Servo: 1-2ms pulse every 20ms = 0-180°
  • Stepper: 200 steps/rev × 16 microsteps = 3200 steps
  • A4988 driver: 1/16 microstepping smooth
  • AccelStepper library: Ramp speed automatically
Robot Reality:

Servo jitters without decoupling cap. Stepper skips steps above 1000 steps/sec without acceleration ramping.

Advanced

IMU Motion Tracking

6/9-axis sensor fusion for drones, robots, VR. Raw accel/gyro → stable orientation.

Orientation Mastery:

  • MPU6050: 3-axis accel + 3-axis gyro
  • Quaternion math eliminates gimbal lock
  • Kalman filter fuses noisy sensor data
  • 100Hz update rate for smooth tracking
Drone Essential:

Raw gyro drifts 1°/sec. Complementary filter (98% accel + 2% gyro) gives stable 0.1° accuracy. Madgwick filter for 9DOF.
